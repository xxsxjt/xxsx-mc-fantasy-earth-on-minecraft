# 魔幻地球 on Minecraft 0.7.0-alpha.5 独立可玩旅程契约

## 目标

把现有启蒙、调律、设施、主动法术、法力耗材和使魔契约串成一条可独立完成的魔幻旅程。它必须体现研究与仪式路线，不能成为玄幻旅程的换色副本，也不能依赖其他生态模组。

## 六个核心里程碑

1. 研读奥术启蒙笔记，真正解锁以太调律。
2. 在面板或专注垫完成一次服务端结算的有效调律。
3. 从炼金台、符文刻台或仪式基座的产物槽取出一次真实加工产物。
4. 成功释放一次当前回路主动法术并支付法力。
5. 成功消耗晶化魔盐或聚魔水晶恢复法力。
6. 完成三次稳定并用使魔契约书绑定一个使魔。

里程碑只由服务端成功路径推进。研究失败、冷却、法力不足、满法力未消耗、客户端点击和未达到稳定度的绑定都不得计数。

## 玩家表面

- 手册首页显示实时完成度、当前里程碑和下一步操作。
- 每个步骤首次完成时只显示一次反馈。
- 六项完成后给出一次使用现有资源的轻量奖励，并引导继续研究护身/呼吸回路。
- 旧存档只静默恢复能从等级和经验证明的步骤。

## 权威与兼容

- 状态保存在 `earth_online_arcana.magic_journey_mask`，遵循双模唯一克隆所有者规则。
- 状态 Payload、Codec、空值、服务端构造和客户端手册必须原子升级。
- 自动化加工不替玩家完成旅程；只有玩家从输出槽取得产物才计入。
- 单独安装时六项都可使用原版材料完成；可选模组只提供额外材料标签与人体恢复联动。

## 验证

1. 位掩码回归覆盖幂等、完成数、完成态和下一步。
2. 六个钩子必须位于真实成功分支。
3. JSON、双语键、Payload 协议和完整构建通过。
4. 仅部署一个 alpha.5 jar，并验证本地、实例和 GitHub 附件哈希。
5. 未获“单独运行”授权时，动作、界面和玩法仍明确标记为待实机验证。
