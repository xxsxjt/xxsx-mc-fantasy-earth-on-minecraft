# 魔幻地球 on Minecraft 大型模组开发计划

## 1. 产品定位

`Fantasy Earth on Minecraft / 魔幻地球 on Minecraft` 复刻大众奇幻、民俗魔法、炼金术、符文、仪式和魔法文明体系。它不是玄幻路线的紫色换皮，也不是只有三台工作台的短流程科技模组。魔幻路线强调以太感知、知识体系、符文语法、材料对应、仪式空间和不同文明对魔法的使用方式。

魔力与玄幻灵力计入同一共享法力值，两条路线可同时学习、上限贡献相加；但魔力来源于以太场、晶体、仪式、符文、炼金和天象，不能直接复用灵脉、经脉和宗门语义。

本模组复刻内容，不复刻繁琐难度。专注垫、法杖、祭坛和学院设施提供效率或新能力，不是玩家打开面板和进行基础调律的硬门槛。

## 2. 不可破坏的兼容边界

### 2.1 原版实体零侵入

- 不向原版实体类注入 Mixin。
- 不修改原版实体属性、AI、方法、掉落和生成。
- 不把现存原版实体替换或转换为魔化实体。
- 不以全局事件永久改写原版生物。

“魔化原版生物”必须注册为独立衍生 `EntityType`。例如符文骷髅可以借用骷髅这一大众原型，但必须拥有本模组独立注册、属性、AI、模型、动画、掉落和生成规则，原版骷髅保持不变。

### 2.2 生态联动

- 与 `earth_online`、`earth_human`、`earth_online_xuanhuan` 通过公开 API、标签和共享协议联动。
- 不直接写相邻模组私有 NBT，不硬绑定对方实现类。
- 缺少可选模组时使用原版退化配方和独立数据，不能阻止启动或创建世界。
- 本仓库只修改自身；发现相邻项目问题时记录，不跨项目直接修复。

## 3. 玩家核心循环

```text
观察异常 -> 记录以太与材料性质 -> 学习学派知识 ->
调律共享法力 -> 炼金/刻写/布置仪式 -> 获得法术与工具 ->
探索生物和遗迹 -> 接触文明聚落 -> 组合或专精魔法路线
```

每项内容都要说明为什么存在、如何获得、在哪里处理、能做什么和如何继续进阶。

## 4. 调律、技能与施法系统

### 4.1 基础调律

- 学习奥术启蒙后，玩家可在任意安全位置打开奥术面板并执行基础调律。
- 基础调律读取当前区块以太场，有冷却并造成局部扰动。
- 专注垫提供稳定回路、效率和恢复增益，是便携辅助，不是必需前置。
- 仪式圈、晶簇、法阵室和天象可以继续强化调律或解锁特殊结果。

### 4.2 快捷键和技能配置

- 提供可配置的“打开奥术面板”快捷键。
- 面板显示共享法力、以太浓度、扰动、来源、冷却、当前回路和专注垫增益。
- 未坐下时可以切换已研究回路并主动调律；坐下时显示离开专注垫操作。
- 后续技能页支持配置法术槽、反应法术、仪式快捷方案、探索技能和生活魔法。

### 4.3 动作和法术表现

- 调律、施法、引导、恢复、刻写和仪式拥有不同动作节奏。
- 法术由服务端判定目标、消耗和结果，客户端只负责输入和表现。
- 粒子形状体现符文、轨迹、节点和元素关系，不只换粒子颜色。
- 高阶法术可以华丽，但必须保留 Minecraft 场景可读性并提供粒子强度配置。

## 5. 学派与知识架构

- 以太学：感知、储存、传导和稳定共享法力。
- 防护学：护盾、环境隔离、抗性和部位保护。
- 塑能学：热、冷、雷、电、动能和光的直接应用。
- 变化学：材料状态、形态和性质的有限改变。
- 召唤与契约：召来独立实体、签订契约和维持消耗。
- 炼金术：溶剂、催化、萃取、结晶、蒸馏和魔性材料。
- 符文学：符号、载体、回路、触发条件和组合语法。
- 仪式学：空间结构、节点、供物、天象和多人协作。
- 占卜学：信息、方向、风险和概率提示，不直接作弊展示全部数据。
- 死灵与生命魔法：作为有代价、有聚落反应的独立路线，而非默认邪恶换皮。

知识由手册条目、研究物、聚落教师、遗迹和实验结果解锁。路线可以交叉，但不能全部压成同一等级条。

## 6. 生物体系

### 6.1 独立衍生实体

- 以太狐、符文狼、晶甲蜘蛛、咒纹骷髅。
- 炼金史莱姆、辉光蝙蝠、元素烈焰体、潮汐守卫。

它们借用大众熟悉的生态原型，但绝不修改或替换原版实体。

### 6.2 原创生物与智慧种族

- 哥布林：拾荒、工坊、贸易和部族路线，不固定为纯敌对。
- 地精：精密装置、测量、符文工程与地下聚落。
- 矮人：采矿、锻造、地下堡垒和材料交易。
- 精灵：林地、自然魔法、长寿知识和生态聚落。
- 兽人：氏族、狩猎、萨满与边境聚落。
- 妖精：小型生态、契约、恶作剧和稀有材料。
- 女巫、法师、炼金师、仪式师、死灵法师、吸血鬼、海洋社会和魔法商队。
- 构装体、元素体、魔像、拟态生物和异界生物。

智慧生物必须有日常行为、生产来源、交易、关系和聚落归属，不能只作为不同皮肤的战斗单位。

## 7. 魔幻聚落与文明

- 魔法学院：学派教师、图书馆、实验室、决斗场和学生生活区。
- 女巫村落：药圃、坩埚、民俗魔法、委托和地方关系。
- 哥布林营地/集市：回收、交易、简易工程和流动商队。
- 地精工坊镇：测量、符文机械、地下管廊和精密零件。
- 矮人山堡：矿山、锻炉、仓库、宴会厅和防御工事。
- 精灵林地城：树木建筑、自然仪式、灵泉和生态守护。
- 兽人氏族营地：狩猎、锻造、萨满和氏族关系。
- 妖精环与微型聚落：季节、花草、契约和空间异常。
- 炼金城市：蒸馏塔、材料市场、研究会和危险废料区。
- 死灵法师墓园/学院：遗体研究、灵魂理论、禁忌交易和声望反应。
- 吸血鬼庄园：夜间社会、血液替代品、政治关系和隐秘任务。
- 水下社会：潮汐祭坛、珊瑚建筑、水下贸易和呼吸魔法。
- 魔法商队与驿站：跨聚落运输、稀有知识和动态事件。
- 废弃法师塔、破损传送站、封印地和异界裂隙：探索型遗迹。

第一批实现女巫村落和哥布林集市，用于验证建筑池、职业、交易、关系和任务；第二批实现学院前哨和矮人工坊。

## 8. 设施、仪式与制作

- 炼金台、符文刻台和仪式基座必须拥有不同交互布局、反馈和处理逻辑。
- 单方块设施是便携或入门版本；完整炼金工坊、仪式圈和魔像装配台使用多方块。
- 仪式结构形成后有独立连接模型和节点效果，任意有效节点可打开核心界面。
- 复杂仪式显示空间条件、节点状态、供物、环境、进度和失败原因。
- JEI 按设施和仪式类型拆分类别，不把全部流程挤进一个页面。

## 9. 美术与界面基准

- 中等精细度 Minecraft 风格，可略华丽，但不做网游面板或写实渲染。
- 魔幻视觉以晶体、玻璃、墨水、羊皮纸、金属回路、星图、木石建筑和几何符文为核心。
- 不把玄幻界面简单换成紫色；魔幻界面强调图谱、节点、回路和研究笔记。
- 不同设施按功能使用不同版式，仍遵守原版容器尺寸和槽位可读性。
- 资源先在最终分辨率联系表检查，再通过实际界面截图和方块六面截图检查。

## 10. 分阶段路线

### 里程碑 A：随身奥术面板（0.4.3）

- 快捷键打开奥术面板。
- 非专注垫基础调律、服务端冷却与区块扰动。
- 专注垫成为明确效率增益。
- 未坐下时面板保持可用，坐下时可离开专注垫。
- 双语、版本、构建、部署和日志验证。

### 里程碑 B：法术配置与表现骨架（0.5.x）

- 法术槽、技能配置、消耗和服务端施法状态机。
- 第一批防护、探索、位移和塑能法术。
- 自定义动作、轨迹粒子、音效和表现降级策略。

### 里程碑 C：生物与契约（0.6.x）

- 3 个独立衍生实体和 3 个原创实体。
- 独立模型、动画、AI、掉落、生成和配置。
- 第一条构装体或契约生物闭环。
- 自动检查不存在原版实体侵入修改。

当前进度（`0.6.0-alpha.1`）：已完成符文守望者首条原创构装体纵向切片，并建立可热重载、带字段校验的聚落目录。其余独立衍生/原创实体、第一条契约闭环和实体聚落生成仍待实现。

### 里程碑 D：聚落与文明（0.7.x）

- 女巫村落、哥布林集市、学院前哨。
- 职业、交易、基础关系、任务和建筑池。
- 聚落产出和炼金/符文/仪式流程相互依赖。

### 里程碑 E：大型内容扩展（0.8+）

- 矮人、精灵、地精、兽人、妖精、亡灵、水下社会等完整路线。
- 学院、法师塔、异界裂隙、大型仪式和法术研究体系。
- 性能预算、数据包 API、配置迁移和正式发布流程。

## 11. 每轮验收标准

- 魔幻与玄幻同时安装时共享法力正确相加，但术语、资源和玩法不互相覆盖。
- 可选依赖缺失时可以独立启动、创建世界并完成基础流程。
- 不存在原版实体 Mixin、属性/AI/掉落/生成修改或实体转换。
- 网络输入由服务端校验，断线、死亡、维度切换和重载不会留下非法施法状态。
- 中英语言键、手册、JEI、tooltip 和界面信息一致。
- 构建和资源检查通过，只部署当前有效 jar，并以新日志确认加载和联网注册。
