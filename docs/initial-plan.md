# Fantasy Earth on Minecraft / 魔幻地球 on Minecraft - Initial Plan

## 1. Positioning

`魔幻地球 on Minecraft` 是独立仓库、独立可玩 mod，不塞进《我的地球》，也不要求安装它。

它的定位不是“现实复刻”，而是提供一套完整的魔幻世界层：本模组与原版材料可以独立支撑魔法仪式、炼金、符文、法器、结界和魔法生态；安装《我的地球》后再接入更丰富的现实矿物、金属、盐类、颜料、玻璃和陶瓷路线。

核心原则：

- 不复刻真实生存难度；`地球人类` 是另一个独立 mod，未来只做联动。
- 不污染 `地球 Online` 本体；本体继续专注真实世界常见内容。
- 不要求玩家懂化学或神秘学；复杂信息放进手册和 JEI，玩法入口保持简单。
- 可独立运行；如果检测到 `earth_online`，就使用更丰富、更真实的材料路线。
- 魔幻风格可以多样，但第一版统一为“现实材料 + 低魔仪式 + 炼金工艺 + 符文工程”。
- 与玄幻分支互通：魔力与灵力统一为法力值，两条路线都练时贡献直接相加。

## 2. 与其他分支的边界

`地球 Online` 本体：

- 真实地质、材料、工业、能源、电力、基础设施、现代和近现代路线。
- 提供矿物、粉末、盐、酸碱、金属、玻璃、陶瓷、纸、颜料、电子材料等基础物质。

`魔幻地球 on Minecraft`：

- 使用上述物质作为魔法材料基础。
- 增加魔力、炼金、符文、仪式、法器、结界、魔法植物/矿物、魔法生态。
- 允许幻想内容，但要保持体系自洽，不变成杂乱魔法包。

`地球人类`：

- 独立 mod，不是附属。
- 未来联动体温、疾病、营养、精神、职业、医疗等真实人物系统。
- 魔幻可以读取人物状态做“诅咒/祝福/精神污染/魔力亲和”，但不在本 mod 内实现真实人体难度。

`修仙`、`科幻`：

- 暂不放入本 mod。
- 修仙偏东方修炼、境界、灵根、丹药、法宝。
- 科幻偏超现实能源、太空、AI、纳米、未来技术。
- 魔幻只做泛奇幻、炼金、符文、仪式、魔法生态。

## 3. First Playable Version

第一版目标不是堆大量法术，而是建立“魔幻材料体系 + 入门机器/设施 + 手册/JEI 查询”的可测试闭环。

MVP 内容：

- 魔力资源：不直接等同 FE/EOU；与玄幻灵力等价为统一法力值。
- 魔幻入门：奥术启蒙笔记，首次使用增加魔力路线法力贡献。
- 入门材料：魔尘、符文墨水、仪式粉笔、晶化魔盐、以太玻璃、符文铜片、聚魔水晶。
- 入门设施：炼金台、符文刻台、仪式基座、聚魔水晶簇、魔法手册。
- 入门玩法：把常见材料转成魔法试剂，再用试剂制作基础护符、符文和工具增强件。
- 查询支持：手册解释路线，JEI 显示炼金/符文/仪式配方。

第一版避免：

- 大量攻击法术。
- 复杂职业/等级系统。
- 大规模维度。
- 过早加入不可控世界污染。
- 强制依赖 `earth_online`。

## 4. Material Routes

Vanilla fallback routes:

- 红石、青金石、石英、紫水晶、荧石、海晶石、末影珍珠、烈焰粉、墨囊、纸、玻璃、铜、金、铁。
- 这些保证没有 `earth_online` 时也能玩。

Earth Online integration routes:

- 石英粉、长石粉、云母粉、盐粉、硫粉、汞滴、金粉、铜线、玻璃配合料、陶瓷绝缘件、颜料、酸碱、稀土、钕铁硼磁材。
- 更真实的材料路线提供更高产率、更稳定配方或高级变体。

示例：

- 石英粉 + 铜线 + 金粉 -> 聚魔线圈。
- 盐粉 + 硫粉 + 颜料 -> 仪式粉笔。
- 墨囊 + 硫酸盐/颜料 + 纸 -> 符文墨水。
- 紫水晶 + 石英粉 + 稀土氧化物 -> 聚魔水晶。
- 玻璃配合料 + 金粉 -> 以太玻璃。

## 5. Systems

### 5.1 Alchemy

炼金不是现代化工重复版，而是魔幻加工接口。

炼金台、符文刻台和仪式基座后续消耗统一法力值，不再为魔幻单独制造一条和玄幻割裂的能量槽。

用途：

- 把现实物质转成魔法试剂。
- 做药剂、粉尘、盐、墨水、媒介。
- 给后续仪式和符文提供材料。

第一版方块：

- 炼金台：单方块，类似工作台/酿造台混合体。
- 炼金坩埚：可选，后续用于液体/热源/多输入。

### 5.2 Runes

符文用于把魔法效果绑定到工具、方块或区域。

第一版方向：

- 采掘符文：小幅提高挖掘效率或自动高亮矿脉。
- 收纳符文：辅助整理、吸附掉落物。
- 保护符文：区域内轻微抗怪或防爆。
- 光照符文：稳定光源或短距照明。

### 5.3 Rituals

仪式用于较大的、多方块结构的魔法行为。

第一版方向：

- 仪式基座 + 粉笔环 + 聚魔水晶。
- 不做复杂 UI 堆叠，优先手册图示和 JEI 分类。
- 成功/失败给玩家明确提示。

### 5.4 Arcane Ecology

魔法生态不应第一版大铺世界生成。

第一版只做少量：

- 聚魔水晶簇。
- 魔化苔藓/符文石。
- 少量可控的魔法植物，用作试剂来源。

## 6. Visual Direction

这不是现代工业风。

视觉关键词：

- 手稿、符文、黄铜、石座、陶瓷、玻璃、晶体、墨水、烛火、星盘、青铜/金箔、矿物颜料。
- 与 `地球 Online` 的现代工业机器拉开风格，但材料来源要能互相解释。
- 方块外观优先可读性：炼金台、符文刻台、仪式基座、聚魔水晶必须一眼能区分。

资产策略：

- 初版可以先用低成本占位图。
- 对正式方块和 GUI，优先使用 `image-2` 生成高质量贴图，再下采样到 32x/64x。
- 每批贴图做 contact sheet 自查，避免假字、风格乱跳和看不出用途。

## 7. Technical Plan

初始实现目标：

- MC 26.2 / NeoForge 26.2.x / Java 25。
- 单模块独立仓库，不直接嵌进 `earth_online`。
- 可选依赖 `earth_online`：检测存在则启用联动配方和材料标签。
- JEI 可选集成：不作为硬依赖。
- 语言：`zh_cn` 和 `en_us` 从第一版开始同步。

建议目录：

```text
earth_online_magic/
├── docs/
├── neoforge-26.2/
├── README.md
└── .gitignore
```

后续真正建 mod 骨架时，优先参考当前 `earth_online/neoforge-26.2` 已验证的 ModDevGradle 结构，但不要复制业务代码。

## 8. First Milestone

第一阶段交付：

- NeoForge 26.2 可编译骨架。
- 创造标签页。
- 魔法手册。
- 炼金台。
- 8-12 个基础魔法材料。
- 6-10 条 vanilla fallback 配方。
- 6-10 条 Earth Online integration 配方。
- JEI 分类：炼金 / 符文 / 仪式。
- 中英语言。

第一阶段验收：

- 无 `earth_online` 时可进游戏。
- 有 `earth_online` 时联动配方出现。
- JEI 可查基础路线。
- 手册能告诉完全不懂技术的玩家第一步做什么。
- 不引入人物难度、体温、疾病、营养等系统。
