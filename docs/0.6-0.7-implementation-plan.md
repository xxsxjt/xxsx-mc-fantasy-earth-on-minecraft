# 魔幻地球 on Minecraft 0.6-0.7 实施计划

## 1. 本轮交付版本

- 目标版本：`0.7.0-alpha.1`
- MC / Loader：Minecraft 26.2 / NeoForge 26.2.0.7-beta
- 部署策略：全部功能、本地双模测试和资源检查完成后一次性部署。
- 写入边界：仅修改 `earth_online_magic`；相邻项目只读。

## 2. 生物生态完成线

### 2.1 独立衍生实体

1. 以太狐：探索型魔法生物，擅长发现以太扰动。
2. 符文狼：护卫型魔法生物，拥有独立符文护甲和战斗 AI。
3. 晶甲蜘蛛：洞穴型魔法生物，使用水晶甲与减速攻击。

### 2.2 原创实体

1. 符文守望者：已有敌对构装体，保留为遗迹/山地威胁。
2. 魔力微灵：可绑定使魔，承担以太感知和回法辅助。
3. 奥术聚落居民：女巫、哥布林商人、学院研究员三种职业变体。

每个实体必须拥有独立 `EntityType`、属性、AI、模型、动画、贴图、掉落、生成蛋或聚落来源，以及中英名称。不得替换或修改原版狐狸、狼、蜘蛛、女巫和村民。

## 3. 使魔绑定闭环

- 新增“使魔契约书”。
- 以魔尘稳定目标，再消耗契约书完成绑定，流程由服务端校验。
- 绑定实体保存主人 UUID，并提供跟随、守护、锚定三种模式。
- 魔力微灵提供以太感知/小额回法；以太狐与符文狼提供不同探索和护卫辅助。
- 绑定不修改玩家原版属性基值，效果使用短时、可移除的状态反馈。
- 保存重载、维度切换和主人离线不会复制或丢失使魔。

## 4. 首批实体聚落

1. 女巫村落：药圃、坩埚庭院、护符屋，生成女巫居民。
2. 哥布林交易所：回收场、估价台、货棚，生成哥布林商人。
3. 魔法学院前哨：研究塔、结界柱、宿舍，生成学院研究员。

实现方式：

- 使用本模组独立 `Feature` 与数据配置，不注入原版村庄结构池。
- 聚落锚点保存类型并在安全的服务端 tick 中生成持久居民。
- 居民使用原版商人容器，交易内容、职业和经济来源属于本模组。
- 矮人深掘聚落与精灵林地聚落继续保留为数据目录，不标记为已生成。

## 5. 美术与资源

- 使用 Image-2 后台生成实体材质母版，保持魔幻的晶体、黄铜、墨水、石材与几何符文语言。
- 不把玄幻实体换色复用；模型轮廓、动画节奏、粒子和材质必须独立。
- 最终 UV、生成蛋、聚落锚点与职业居民在客户端实际资源重载中检查。

## 6. 验收标准

- 6 个实体均可生成并读取正确属性。
- 自然生成、敌对生成和聚落来源注册完整。
- 使魔绑定、模式切换和持久化通过服务端测试。
- 三类聚落可在新世界生成且居民可交易。
- 仅安装本模组时服务端到 `Done`，可从原版材料进入魔法、设施和使魔闭环。
- 与《玄幻地球 on Minecraft》同时加载时服务端到 `Done`，共享法力不覆盖彼此数据。
- 客户端完成资源重载，日志无本模组引起的 `ERROR/FATAL`。
- 仅在全部通过后部署、推送 GitHub 和发布测试版。
