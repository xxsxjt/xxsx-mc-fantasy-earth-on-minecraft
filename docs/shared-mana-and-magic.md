# 共享法力值与魔幻调律协议

## 核心决定

`魔幻地球 on Minecraft` 不再把魔力做成与玄幻灵力割裂的第二条能量。两个独立模组同时安装时，魔力和灵力统一为“法力值”，但来源、训练方式和内容风格不同。

```text
法力上限 = 基础法力
        + 玄幻路线贡献
        + 魔幻路线贡献
        + 装备贡献
        + 临时效果贡献
```

玩家同时学习魔幻和玄幻时，两个贡献直接相加。魔幻不覆盖玄幻，玄幻也不覆盖魔幻。

## 共享持久数据与所有权

本分支与玄幻分支约定使用：

```text
earth_online_arcana.current_mana
earth_online_arcana.base_mana
earth_online_arcana.xuanhuan_mana_bonus
earth_online_arcana.magic_mana_bonus
earth_online_arcana.equipment_mana_bonus
earth_online_arcana.temporary_mana_bonus
earth_online_arcana.qi_absorption_rate
earth_online_arcana.magic_attunement_rate
earth_online_arcana.cultivation_level
earth_online_arcana.magic_research_level
```

魔幻分支写入自身研究等级、经验、调律率与法力贡献，并读取统一法力总量用于显示和消耗。它不得重置玄幻路线的数据。

双模并装时：

- 玄幻模块负责复制 `earth_online_arcana.*` 克隆数据并消费共享身体适应效果。
- 魔幻模块检测到玄幻模块后让出这两项职责，避免重复复制与重复减耗、防护或呼吸结算。
- 单独安装魔幻模块时，魔幻模块自行保留共享数据并提供完整被动收益。

## 魔幻入门

三类入口必须保持分离：

- `魔幻田野手册`：由一块泥土、任意木板或任意石头合成，只负责教学、实时状态和下一步，不参与研究配方。
- `奥术启蒙笔记`：由书、魔尘和紫水晶碎片合成，首次右键只负责解锁基础研究。
- 护身与呼吸札记：要求先完成奥术启蒙，首次右键只解锁对应回路。
- 调律面板与专注垫：重复成长入口，负责环境读取、冷却、法力恢复、经验和升级。
- 主动法术：通过独立按钮或快捷键释放，服务端校验研究、法力和冷却。

## 与玄幻分支联动

玩家同时安装两个独立模组时，推荐起步体验：

1. 分别用泥土、木板或石头取得两本低成本手册，了解两条独立路线。
2. 按各自配方制作《引气诀》和《奥术启蒙笔记》，分别解锁灵力与魔力贡献。
3. 在修炼、调律面板和手册状态页查看统一法力值；两条路线贡献相加，但训练方法、技能和表现保持独立。

## 当前扩展方向

1. 手册、面板和 JEI 共用真实等级、经验、冷却、配方与仪式谓词。
2. 炼金、符文刻写、仪式和主动法术使用不同交互与视觉语言，不做通用机器界面换皮。
3. 仪式结构和以太场按区块提供可诊断的加成与干扰，同时保留无可选模组时的独立路线。
4. 共享键是当前兼容协议；稳定后再抽取版本化公共 API，不增加强制前置。
5. 魔幻与玄幻只统一法力总量，不统一研究树、动作、材料、设施或美术风格。
